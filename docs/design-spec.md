# 设计规范 — ML 刷题软件

## 设计原则

1. **简洁**：每屏只展示当前必要的信息，避免视觉噪音
2. **直观**：交互方式符合用户直觉，点击即所得
3. **友好**：淡蓝主色调带来平静、专业的感觉，适合学习场景
4. **响应**：所有操作有即时视觉反馈（hover、active、状态变化）

## 交互规范

### 概念题
1. 用户看到 4 个选项卡片
2. 点击任意选项 → 该选项高亮（蓝色边框 + 淡蓝背景），取消之前的选择
3. "提交答案"按钮激活（默认禁用，直到选了选项）
4. 提交后：正确选项变绿，如选错则选中的变红
5. 显示反馈区 + 解析文字
6. "下一题"按钮出现

### 代码拼图题
1. 上方"代码块池"显示所有块（正确 + 迷惑），随机排列
2. 下方"你的答案"区域为空
3. 用户点击池中的块 → 块从池中消失（变灰虚化），按点击顺序出现在答案区
4. 用户点击答案区中的块 → 块从答案区移除，回到池中（恢复可选）
5. "提交答案"按钮在选择至少 1 个块后激活
6. 提交后：位置正确的块变绿，错误的块变红
7. 显示正确排序 + 解析

### 过渡动画
- 屏幕切换：淡入 + 上移 8px（0.3s）
- 按钮点击：缩放 97%（按下反馈）
- 进度条：宽度过渡 0.4s
- 卡片/选项 hover：边框颜色过渡 0.2s

## 响应式设计

| 断点 | 行为 |
|------|------|
| > 860px | 卡片最大宽度 860px，居中 |
| 500-860px | 卡片填满宽度（留 16px margin） |
| < 500px | 卡片内边距缩减，筛选按钮缩小，错题统计改为单列 |
| < 400px | 最小支持宽度，代码块允许横向滚动 |
