# 分阶段开发执行步骤 — ML 刷题软件

## 开发原则

1. **一次只做一件事**：每个阶段聚焦一个模块，完成并验证后再进入下一阶段
2. **可验证的交付**：每阶段结束后有明确的验收标准
3. **稳健推进**：不跳步，不并行处理会影响稳定性的任务
4. **记录留痕**：每天在 `dev-log/` 中记录进度
